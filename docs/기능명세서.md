## 2. 시스템 구성 요약

- **프론트엔드**: React + TypeScript + Vite, Tailwind CSS, Zustand, React Router v6
- **백엔드**: Node.js + Express (프론트는 Supabase에 직접 접근하지 않고 백엔드를 경유)
- **DB**: Supabase (Postgres) — `topics`, `interview_results`
- **외부 API**: OpenAI Speech-to-Text(Whisper), GPT 분석
- **페이지 구성**: Home → Interview → Result (React Router)

## 3. 화면별 기능 명세

### 3.1 Home 페이지 (`/`)

| ID | 기능명 | 설명 |
|---|---|---|
| FR-HOME-01 | 서비스 소개 표시 | 서비스명, 소개 문구 노출 |
| FR-HOME-02 | 시작하기 버튼 | 클릭 시 Interview 페이지로 이동하며 랜덤 Topic+질문 세트 요청 트리거 |

**처리 흐름**
1. 사용자가 "시작하기" 클릭
2. `GET /api/questions/random` 호출
3. 응답을 Zustand 스토어에 저장 (`topic`, `questions[]`)
4. `/interview` 라우트로 이동

**예외 처리**
- API 실패/네트워크 오류 시 에러 메시지 노출, "다시 시도" 버튼 제공 (화면 전환 안 함)

### 3.2 Interview 페이지 (`/interview`)
메인 페이지랑 인터뷰 페이지 꾸미는 것 까지 일단 해보기

| ID | 기능명 | 설명 |
|---|---|---|
| FR-INT-01 | Topic/질문 표시 | 현재 질문 인덱스(1~3)에 해당하는 질문 텍스트와 Topic 카드 표시 |
| FR-INT-02 | 음성 녹음 시작/중지 | 브라우저 MediaRecorder 기반 녹음, 상태 표시(대기/녹음중/완료) |
| FR-INT-03 | 녹음 권한 처리 | 마이크 권한 요청 및 거부 시 안내 메시지 |
| FR-INT-04 | 다음 질문 이동 | 현재 질문 녹음 완료 후 다음 질문으로 전환 (질문 1→2→3) |
| FR-INT-05 | 평가하기 | 3문제 녹음 완료 후 활성화되는 버튼, 클릭 시 STT+분석 파이프라인 실행 |
| FR-INT-06 | 진행 상태 표시 | 현재 몇 번째 질문인지(예: 2/3) 시각적 표시 |

**입력/출력**
- 입력: 사용자 음성(브라우저 녹음, 오디오 Blob)
- 출력: 질문별 오디오 Blob → STT 텍스트(임시 상태 저장, 로컬 상태에서만 보관) -> 브라우저에 임시저장해놓고, 저장한것을 다시 들을 수 있는정도로만 일단 구성하기. + Main page 꾸미기 + 로그인 누르면 바로 인터뷰 페이지 넘어갈 수 있도록 만들기/

**처리 흐름 (질문 1건 기준)**
1. "녹음 시작" 클릭 → `useAudioRecorder` 훅으로 녹음 시작
2. "녹음 종료" 클릭 → Blob 생성, 로컬 상태에 저장
3. 다음 질문 존재 시 인덱스 증가 후 1로 복귀
4. 질문 3까지 완료되면 "평가하기" 버튼 활성화

**"평가하기" 클릭 시 처리 흐름**
1. 각 질문 오디오 Blob을 `POST /api/stt` 로 순차/병렬 전송 → 텍스트 3개 확보
2. 변환된 텍스트를 화면에 확인용으로 표시 (선택적, 즉시 다음 단계로 진행 가능)
3. `POST /api/analyze` 에 `{ topic, answers: [a1, a2, a3] }` 전송
4. 응답(`id, level, strengths, improvements, repeatedPhrases, suggestions, sampleAnswer, disclaimer`)을 스토어에 저장
5. `/result` 라우트로 이동
6. 음성 원본 Blob은 STT 변환 완료 즉시 폐기(메모리에서 제거), 저장하지 않음

**예외 처리**
- 마이크 권한 거부: "마이크 권한이 필요합니다" 안내, 재요청 유도
- 녹음 없이 다음 질문/평가 시도: 버튼 비활성화로 방지
- STT 실패: 해당 질문 재녹음 유도 메시지, "다시 시도" 버튼
- 분석 API 실패(타임아웃/5xx): 에러 메시지 + "다시 시도" 버튼, 페이지 이동 안 함
- 네트워크 오류: 재시도 제안 메시지

## 4. API 기능 명세

### 4.1 `GET /api/questions/random`

| 항목 | 내용 |
|---|---|
| 설명 | Supabase `topics` 테이블에서 랜덤 1건 조회 |
| 요청 | 파라미터 없음 |
| 응답 200 | `{ topic: string, questions: [{ id, text }, ...] }` (3개 질문) |
| 오류 | DB 조회 실패 시 500, `{ error: message }` |
| 비고 | 매 호출마다 랜덤 선택 로직 필요 |

### 4.2 `POST /api/stt`

| 항목 | 내용 |
|---|---|
| 설명 | 음성 파일을 텍스트로 변환 (OpenAI Whisper) |
| 요청 | `multipart/form-data`, 필드명 `audio` |
| 응답 200 | `{ text: string }` |
| 검증 | 파일 존재 여부, 허용 포맷/용량 제한 |
| 오류 | 파일 누락/포맷 오류 시 400, OpenAI 오류 시 502/500 |
| 데이터 보관 정책 | 변환 완료 즉시 업로드된 음성 파일 삭제(디스크/메모리 미보관), Supabase Storage 미사용 |

### 4.3 `POST /api/analyze`

| 항목 | 내용 |
|---|---|
| 설명 | 3개 답변 텍스트에 대한 GPT 분석 및 레벨 산출, 결과 Supabase 저장 |
| 요청 | `{ topic: string, answers: [string, string, string] }` |
| 응답 200 | `{ id, level, strengths[], improvements[], repeatedPhrases[], suggestions[], sampleAnswer, disclaimer }` |
| 처리 로직 | 1) 프롬프트 조립(질문+답변) → 2) GPT 호출 → 3) JSON 파싱/검증 → 4) `interview_results` insert → 5) 응답 반환 |
| 검증 | `answers` 3개 모두 비어있지 않은지 확인 |
| 오류 | 입력 누락 400, GPT 응답 파싱 실패 500, GPT/DB 오류 502/500 |
| 필수 포함 | 응답에 면책 문구(`disclaimer`) 항상 포함 |

## 5. 데이터 모델 (Supabase)

### 5.1 `topics`

| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | uuid, PK | 고유 ID |
| topic | text | 토픽명 |
| question_1/2/3 | text | 질문 3종 |
| created_at | timestamptz | 생성 시각 |

### 5.2 `interview_results`

| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | uuid, PK | 세션 ID |
| topic_id | uuid, FK | `topics.id` 참조 |
| answer_1/2/3 | text | STT 변환 답변 |
| level | text | 예상 OPIC 레벨 |
| strengths | jsonb | 강점 목록 |
| improvements | jsonb | 개선점 목록 |
| repeated_phrases | jsonb | 반복 표현 |
| suggestions | jsonb | 추천 표현 |
| sample_answer | text | 모범 답변 |
| created_at | timestamptz | 생성 시각 |

- `user_id` 없음 (비회원 세션 단위)
- 음성 원본은 어떤 테이블/스토리지에도 저장하지 않음

## 6. 상태 관리 명세 (Zustand)

| 상태 | 설명 |
|---|---|
| `topic` | 현재 세션의 Topic명 |
| `questions` | 질문 3개 배열 `{id, text}[]` |
| `currentQuestionIndex` | 현재 진행 중인 질문 인덱스 (0~2) |
| `recordings` | 질문별 오디오 Blob (STT 변환 후 폐기) |
| `sttResults` | 질문별 STT 변환 텍스트 |
| `analysisResult` | `/api/analyze` 응답 전체 |
| `status` | 로딩/에러 상태 (녹음중, 변환중, 분석중, 완료, 오류) |

## 7. 공통 예외 처리 정책

| 상황 | 처리 |
|---|---|
| 마이크 권한 거부 | 안내 메시지 + 브라우저 설정 안내 |
| 네트워크 오류 | "다시 시도" 버튼, 이전 화면 상태 유지 |
| STT 변환 실패 | 해당 질문 재녹음 유도 |
| GPT 분석 실패/타임아웃 | 에러 메시지 + 재시도 버튼, Result 페이지 이동 차단 |
| 서비스 전체 장애 | 기본 텍스트 피드백 안내(대체 메시지) |
| 결과 없이 Result 접근 | Home으로 리다이렉트 |

## 8. 비기능 요구사항 반영

| 항목 | 기준 |
|---|---|
| 응답 속도 | STT+분석 전체 10초 이내 목표 |
| 가용성 | 99% 이상 |
| 보안 | OpenAI API 키·Supabase service role key는 백엔드 `.env`에서만 사용, 클라이언트 미노출 (`VITE_` 접두사 변수만 프론트 노출) |
| 데이터 보존 | 음성 원본 미저장, 답변 텍스트/분석 결과만 Supabase 저장 |
| 브라우저 호환성 | 최신 Chrome/Edge/Firefox |
| 접근성 | 기본 웹 접근성(포커스 이동, 대체 텍스트 등) 준수 |

## 9. MVP 범위 (기능명세 기준)

**포함**
- FR-HOME-01, 02
- FR-INT-01 ~ 06
- FR-RES-01 ~ 08
- API 3종 (`questions/random`, `stt`, `analyze`)
- Supabase `topics`, `interview_results` 저장/조회

**제외 (Phase 2 이후)**
- 회원가입/로그인, 사용자 프로필
- 기록 조회 UI(마이페이지)
- 통계 대시보드
- 발음/억양/속도 분석
- 결제, 관리자 기능
